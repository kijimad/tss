# 仮想端末エミュレータ (VTerm)

ブラウザ上で動作する VT100 互換ターミナルエミュレータ。PTY (疑似端末)、VT100 エスケープシーケンスパーサー、スクリーンバッファ、Canvas レンダラーを全て TypeScript で実装。右パネルで PTY の I/O とエスケープシーケンスの解釈がリアルタイムで見える。

## 起動

```bash
npm install
npm run dev
```

## 画面の見方

- **左**: Canvas ベースのターミナル画面 (80x24)。文字色、背景色、太字、下線、反転、カーソル点滅を描画
- **右**: **PTY / Escape Sequence Trace** — キーボード入力のバイト列、シェル出力、シグナル、エスケープシーケンスの解釈がリアルタイム表示

## アーキテクチャ

```
キーボード → [PTY master] → Line Discipline → [PTY slave] → Shell
                                  ↕ echo
Canvas ← [VT100 Parser] ← [PTY master] ← [PTY slave] ← Shell 出力
```

### PTY (疑似端末)
- **マスター側**: ターミナルエミュレータが接続。キーボード入力を書き込み、画面出力を読み取る
- **スレーブ側**: シェルが接続。入力を読み取り、出力を書き込む
- **Line Discipline**: エコー、行バッファリング、Ctrl+C シグナル処理

### VT100 パーサー
エスケープシーケンスを解釈して 80x24 のセル配列を更新する。

### Canvas レンダラー
セル配列を Canvas に描画。各セルの文字・色・属性をピクセルに変換。

## コマンド

| コマンド | 説明 |
|---------|------|
| `echo [-e] <text>` | テキスト出力 (`-e` でエスケープシーケンス解釈) |
| `cat <file>` | ファイル内容表示 |
| `ls [-l] [dir]` | ディレクトリ一覧 (色付き) |
| `cd <dir>` | ディレクトリ移動 |
| `pwd` | カレントディレクトリ |
| `mkdir <dir>` | ディレクトリ作成 |
| `touch <file>` | ファイル作成 |
| `rm <file>` | ファイル削除 |
| `clear` | 画面クリア (ESC[2J) |
| `colors` | 16色カラーパレット表示 |
| `demo` | テキスト属性デモ（太字、下線、反転、色） |
| `history` | コマンド履歴 |
| `env` | 環境変数一覧 |
| `export K=V` | 環境変数設定 |
| `tput cols/lines` | ターミナルサイズ |
| `Ctrl+C` | SIGINT (コマンド中断) |
| `Ctrl+D` | EOF (空行で入力終了) |

## 実験

### 実験 1: 色付き出力

```
echo -e "\e[31mRed\e[0m \e[32mGreen\e[0m \e[34mBlue\e[0m"
```

ターミナル画面に赤・緑・青のテキストが表示される。右パネルに SGR エスケープシーケンスの解釈が表示される:
```
ESC: SGR 31 — fg=1 bg=-1 bold=false
ESC: SGR 0 — fg=-1 bg=-1 bold=false
```

---

### 実験 2: 16色カラーパレット

```
colors
```

前景色 0-15 と背景色 0-15 を全て表示。ターミナルがサポートする色を確認できる。

---

### 実験 3: テキスト属性

```
demo
```

太字、下線、反転、色付き太字が一行で表示される。Canvas レンダラーがこれらの属性を正しく描画しているか確認。

---

### 実験 4: PTY の動作を観察する

右パネルを見ながら1文字ずつ入力する:

```
hello
```

1文字ごとに:
1. `kbd -> pty: "h" [68]` — キーコード 0x68 が PTY マスターに書き込まれる
2. `echo: 'h'` — Line Discipline がエコーバック
3. 画面に `h` が表示される
4. Enter で `line: "hello"` — 行バッファがシェルに送られる

---

### 実験 5: Backspace の動作

`helo` と入力して Backspace を2回押し、`llo` と入力:

右パネルに:
```
kbd → pty: "\x7f" [7f]    ← Backspace キーコード
```
画面では文字が消えて `hello` と表示される。PTY の Line Discipline が `\b \b` (バックスペース + スペース + バックスペース) を送って画面から文字を消す。

---

### 実験 6: Ctrl+C (SIGINT)

何か入力中に Ctrl+C:

右パネルに:
```
SIGNAL: SIGINT (Ctrl+C)
```
行バッファがクリアされて新しいプロンプトが表示される。

---

### 実験 7: 画面クリア

```
clear
```

`ESC[2J` (画面全体クリア) + `ESC[H` (カーソルを原点に) が送られる。右パネルに:
```
ESC: CSI 2J — clear screen
ESC: CSI H — Cursor to (0,0)
```

---

### 実験 8: ファイル操作

```
ls
cat hello.txt
echo "new content" > test.txt
cat test.txt
ls -l
```

ディレクトリは青、実行可能ファイル(.sh)は緑で表示される。`-l` で詳細表示。

---

### 実験 9: プロンプトのエスケープシーケンス

プロンプト `user@host:~$` は以下のエスケープシーケンスで描画される:
```
\e[1;32m  → 太字+緑 (ユーザ名@ホスト名)
\e[0m     → リセット
\e[1;34m  → 太字+青 (ディレクトリ)
\e[0m     → リセット
```

右パネルの `slave_write` で実際のバイト列が見える。

---

### 実験 10: Canvas レンダリングの確認

```
echo -e "\e[41m\e[37m White on Red \e[0m"
echo -e "\e[7m Inverse \e[0m"
echo -e "\e[4m Underline \e[0m"
```

背景色(41=赤)、反転(7)、下線(4) がそれぞれ Canvas 上で正しく描画される。

## VT100 エスケープシーケンス一覧

| シーケンス | 説明 |
|-----------|------|
| `ESC [ n A` | カーソル上に n 行移動 |
| `ESC [ n B` | カーソル下に n 行移動 |
| `ESC [ n C` | カーソル右に n 列移動 |
| `ESC [ n D` | カーソル左に n 列移動 |
| `ESC [ n ; m H` | カーソルを (n, m) に移動 |
| `ESC [ 2 J` | 画面クリア |
| `ESC [ K` | 行末までクリア |
| `ESC [ 0 m` | 属性リセット |
| `ESC [ 1 m` | 太字 |
| `ESC [ 4 m` | 下線 |
| `ESC [ 7 m` | 反転 |
| `ESC [ 30-37 m` | 前景色 (黒,赤,緑,黄,青,紫,水,白) |
| `ESC [ 40-47 m` | 背景色 |
| `ESC [ 90-97 m` | 明るい前景色 |

## 実際のターミナルとの違い

| xterm/kitty/alacritty | このシミュレータ |
|---------------------|-----------------|
| OS の PTY デバイス (/dev/ptmx) | TypeScript の双方向パイプ |
| fork+exec でシェル起動 | 関数呼び出し |
| OpenGL/GPU レンダリング | Canvas 2D |
| 256色 + TrueColor | 16色 |
| Unicode 全サポート | ASCII のみ |
| マウスイベント | 未対応 |
| 選択 + コピー | 未対応 |
| スクロールバック | 未対応 |
